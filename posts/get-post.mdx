💡 **Summary**

- GET과 POST는 모두 HTTP 요청 메서드로 서버와 통신하는데 사용
- GET은 정보를 조회하며, POST는 서버에 정보를 전송하기 위해 사용
</aside>

## 사용자 인터페이스와 서버의 소통 방법

1. **HTTP 요청**
    - 대표적인 메서드로 :  GET, POST, PUT, DELETE
2. **AJAX (Asynchronous JavaScript and XML)**
    - 웹 페이지의 일부분만 업데이트하기 위해 사용하는 기술
        
        사용자가 웹 페이지의 특정 부분에서 버튼을 클릭하거나 입력을 할 때, **전체 웹 페이지를 다시 로드하지 않고도 필요한 데이터만 서버에서 받아올 수 있음**
        
    - AJAX 요청할 때, GET이나 POST 등의 HTTP 메서드를 사용하여 서버와 통신
    - **예시**
        1. 사용자가 검색창에 '고양이' 입력 
        2. 사용자가 검색 버튼을 클릭
        3. 웹 페이지 전체를 새로고침하지 않고, 백그라운드에서 서버에 '고양이' 검색 결과를 요청
        4. 서버는 '고양이'에 관련된 검색 결과를 웹 페이지에 전송
        5. 웹 페이지는 받은 검색 결과를 사용자에게 표시
            - 페이지 전체를 다시 로드하지 않고도 검색 결과만 변경
3. **WebSockets**
    - 지속적, 양방향 통신 가능
    - ex) 실시간 채팅 등
4. **API (Application Programming Interface)**
    - 클라이언트(UI)가 API에 따라 서버와 통신
        - API : 서버가 제공하는 특정한 포맷과 규칙
5. **서버 응답**
    - 클라이언트(UI)는 받아온 데이터를 처리하여 사용자에게 적절한 형태로 표시
    - ex) JSON 형식의 게시물 리스트 데이터를 받아와서 웹 페이지에 게시물 목록으로 렌더링
        
        (서버 : 대개 JSON, XML, HTML 등의 형식으로 데이터를 응답) 
    - **(+) HTTP 요청 vs AJAX ?**
        - HTTP 요청
            - 웹 클라이언트와 서버 사이의 통신 규약
            - 모든 웹 통신은 HTTP 요청을 통해 이루어짐
        - AJAX
            - HTTP 요청 중 일부를 선택적으로, 비동기적으로 처리하기 위한 기술
            - 웹 페이지의 특정부분만 업데이트

## GET vs POST

### Get

- 웹 페이지나 리소스를 가져오기 위해 사용
    
    ex) 데이터 요청, 웹 페이지 로딩, 이미지/스타일시트 로딩 등
    
- URL에 GET요청 파라미터 포함됨
    - `example.com/search?q=keyword`
    - 암호, 등과 같이 민감한 정보 전송에는 적합 X
- **AJAX** : AJAX를 통해 서버에서 데이터를 비동기적으로 가져올때 사용!

### Post

- 서버에 새로운 데이터를 제출 / 서버의 상태를 변경할 때 사용
    
    ex) 회원 가입 폼 제출, 댓글 작성, 파일 업로드 등
    
- 전송 데이터의 크기에 거의 제한이 없음
( **∵** URL에 데이터를 첨부 X)
- **브라우저 히스토리에 데이터 없이 저장** ⇒ 새로고침/재방문 시 재전송 X
- **AJAX**: 페이지 새로고침 없이 비동기 통신(서버 상태 변경/새로운 데이터 전송 가능)
- **(+) Post 요청이 브라우저 히스토리에 데이터 내용과 함께 저장된다면??**
    
    사용자가 페이지를 새로고침하거나 페이지를 재방문할 때마다 그 **`POST`** 요청이 자동으로 재전송
    
    → 같은 댓글이 반복적으로 작성 / 회원가입 폼 여러번 전송 ..등등 
    
    ⇒ 즉, POST 요청에는 “어떤 정보를 보냈는지”를 저장하지 않고 **“정보를 보냈다”는 사실만 히스토리에 저장**

💡 **Wrap-up**

- 웹 페이지와 서버는 HTTP 요청, AJAX, WebSockets, API 등의 방법으로 소통
- **GET과 POST**는 모두 HTTP 요청 메서드로 서버에 데이터를 요청하거나 전송하는데 사용
    - **GET**은 서버로부터 정보를 조회하기 위해 사용
    - **POST**는 정보를 서버에 전송하기 위해 사용
